## Documentation Standards

### Structure
- **Overview**: High-level purpose and context
- **Architecture**: How it works
- **Usage**: How to use it
- **Examples**: Code samples
- **Best Practices**: Recommendations
- **Troubleshooting**: Common issues and solutions
- **Related Documentation**: Links to other docs

### Code Examples
- Include imports
- Show complete, runnable code
- Annotate with comments
- Reference source file locations

### Updates
- Update "Last Updated" date when modifying
- Add version numbers for breaking changes
- Cross-reference related documentation
- Keep examples in sync with codebase

## Contributing to Documentation

### Adding New Documentation
1. Create file in `/docs/` directory
2. Follow existing documentation structure
3. Add entry to this index
4. Cross-reference related docs
5. Update README.md if major addition

### Updating Existing Documentation
1. Verify information accuracy
2. Update examples if code changed
3. Update "Last Updated" date
4. Test any code examples
5. Commit with descriptive message

### Documentation Review
- Accuracy: Information is correct
- Completeness: All aspects covered
- Clarity: Easy to understand
- Examples: Working code samples
- Links: All references valid
